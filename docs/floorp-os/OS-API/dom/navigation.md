## 待機

### 要素の待機

要素が指定の状態になるまで待機する。

**エンドポイント**: `POST /instances/:id/waitForElement`

**リクエスト**:

```json
{
  "selector": ".loading-complete",
  "state": "visible",
  "timeout": 10000
}
```

| パラメータ    | 必須     | 型                                                        | デフォルト   | 説明                                                                     |
| ------------- | -------- | --------------------------------------------------------- | ------------ | ------------------------------------------------------------------------ |
| `selector`    | 条件付き | `string`                                                  | —            | CSS セレクタ。                                                           |
| `fingerprint` | 条件付き | `string`                                                  | —            | 要素フィンガープリント。`state:"attached"` との組み合わせは 400 エラー。 |
| `state`       | いいえ   | `"attached"` \| `"visible"` \| `"hidden"` \| `"detached"` | `"attached"` | 待機する状態。                                                           |
| `timeout`     | いいえ   | `number`                                                  | `5000`       | タイムアウト（ミリ秒）。0〜60000 にクランプされる。                      |

**成功レスポンス** (200):

```json
{
  "ok": true,
  "found": true
}
```

**タイムアウト時レスポンス** (200):

```json
{
  "ok": true,
  "found": false
}
```

**失敗レスポンス** (400):

```json
{
  "error": "fingerprint with state 'attached' is not supported"
}
```

**内部動作**:

1. `MutationObserver` を設定する。
2. 指定された `state` に応じて監視する:
   - `attached`: 要素が DOM に追加されるのを待つ。
   - `visible`: 要素が表示状態になるのを待つ。
   - `hidden`: 要素が非表示になるのを待つ。
   - `detached`: 要素が DOM から削除されるのを待つ。
3. タイムアウトに達した場合、`found: false` を返却する。
4. デッドオブジェクトが検出された場合は `false` を返却する。

### ドキュメント準備完了の待機

ドキュメントの準備完了を待機する。

**エンドポイント**: `POST /instances/:id/waitForReady`

**リクエスト**:

```json
{
  "timeout": 15000
}
```

| パラメータ | 必須   | 型       | デフォルト | 説明                                                |
| ---------- | ------ | -------- | ---------- | --------------------------------------------------- |
| `timeout`  | いいえ | `number` | `15000`    | タイムアウト（ミリ秒）。0〜60000 にクランプされる。 |

**成功レスポンス** (200):

```json
{
  "ok": true
}
```

**失敗レスポンス** (500):

```json
{
  "error": "string"
}
```

**内部動作**:

1. `document.readyState` を確認する。
2. `readystatechange` イベントと `DOMContentLoaded` イベントを監視する。
3. `readyState` が `"interactive"` または `"complete"` になったら完了する。

### ネットワークアイドルの待機

ネットワークリクエストが落ち着くまで待機する。

**エンドポイント**: `POST /instances/:id/waitForNetworkIdle`

**リクエスト**:

```json
{
  "timeout": 5000,
  "maxInflight": 0,
  "idleDuration": 500,
  "ignorePatterns": ["analytics", "tracking"]
}
```

| パラメータ       | 必須   | 型         | デフォルト | 説明                                     |
| ---------------- | ------ | ---------- | ---------- | ---------------------------------------- |
| `timeout`        | いいえ | `number`   | `5000`     | タイムアウト（ミリ秒）。                 |
| `maxInflight`    | いいえ | `number`   | —          | アイドルと判定する最大同時リクエスト数。 |
| `idleDuration`   | いいえ | `number`   | —          | アイドル判定までの安定期間（ミリ秒）。   |
| `ignorePatterns` | いいえ | `string[]` | —          | 無視する URL パターンの配列。            |

**成功レスポンス** (200):

```json
{
  "ok": true
}
```

**失敗レスポンス** (500):

```json
{
  "error": "string"
}
```

## ナビゲーション

### ページ遷移

指定した URL に遷移する。

**エンドポイント**: `POST /instances/:id/navigate`

**リクエスト**:

```json
{
  "url": "https://example.com/page"
}
```

| パラメータ | 必須     | 型       | デフォルト | 説明                                                    |
| ---------- | -------- | -------- | ---------- | ------------------------------------------------------- |
| `url`      | **はい** | `string` | —          | 遷移先の URL。`http`、`https`、`about:blank` のみ許可。 |

**成功レスポンス** (200):

```json
{
  "ok": true
}
```

**失敗レスポンス** (400):

```json
{
  "error": "url required"
}
```

**内部動作**:

1. URL のバリデーション（`http`/`https`/`about:blank`）。
2. ブラウザに URL をロードする。
3. **通常のナビゲーション**: `STATE_STOP` イベントを待機する。
4. **SPA ナビゲーション**: `LOCATION_CHANGE_SAME_DOCUMENT` イベントを検知した場合、追加で 500ms 待機する。

**備考**: SPA の場合はヒストリー API によるナビゲーションを検知し、レンダリングが安定するまで 500ms の追加待機を行う。

### 現在の URI の取得

インスタンスの現在の URL を取得する。

**エンドポイント**: `GET /instances/:id/uri`

**パスパラメータ**:

| パラメータ | 必須     | 型       | 説明              |
| ---------- | -------- | -------- | ----------------- |
| `id`       | **はい** | `string` | インスタンス ID。 |

**リクエスト**: パラメータなし。

**成功レスポンス** (200):

```json
{
  "uri": "https://example.com/current-page"
}
```

**タブが存在しない場合** (200):

```json
{
  "uri": null
}
```
